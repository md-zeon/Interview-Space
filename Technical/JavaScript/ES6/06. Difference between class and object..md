# Difference between class and object

Understanding the difference between classes and objects is fundamental to object-oriented programming. While the concepts are similar, there are key distinctions, especially in how JavaScript implements them.

## Class vs Object

### **Class**
- A **blueprint or template** for creating objects
- Defines the structure, properties, and behaviors that objects of that class should have
- Cannot be used directly; must be instantiated
- Describes "what" an object should be (structure and capabilities)

**Example:**
```javascript
// Class definition (ES6)
class Car {
  constructor(make, model, year) {
    this.make = make;
    this.model = model;
    this.year = year;
  }

  start() {
    return `${this.make} ${this.model} is starting...`;
  }

  drive() {
    return `${this.make} ${this.model} is driving...`;
  }
}
```

### **Object**
- A **concrete instance** created from a class or created directly
- Represents a specific entity with actual data values
- Can be used directly and manipulated
- Implements "who" or "what" something specifically is

**Example:**
```javascript
// Object creation (instance of class)
const myCar = new Car('Toyota', 'Camry', 2020);

console.log(myCar.make); // 'Toyota'
console.log(myCar.start()); // 'Toyota Camry is starting...'

// Or object literal
const anotherCar = {
  make: 'Honda',
  model: 'Civic',
  year: 2019,
  start: function() {
    return `${this.make} ${this.model} is starting...`;
  }
};
```

## Key Differences

### **01. Instantiation**
- **Class**: Needs to be instantiated using `new` keyword
- **Object**: Can exist independently or be created from classes

### **02. Reusability**
- **Class**: Can create multiple objects with the same structure
- **Object**: Each object can be unique, even if created from same class

### **03. Memory**
- **Class**: Template (methods stored once on prototype)
- **Object**: Actually occupies memory with data

### **04. Modification**
- **Class**: Structure rarely changes after definition
- **Object**: Properties can be added/removed dynamically

## In JavaScript Context

### **Classes in JavaScript (ES6+)**
- Introduced in ES6 as syntactic sugar over prototype-based inheritance
- Uses `class` keyword but still works via prototypes under the hood
- Provides cleaner syntax for constructor functions and inheritance

**Class Equality:**
```javascript
class PersonClass {
  constructor(name) {
    this.name = name;
  }
  greet() { return `Hello, ${this.name}`; }
}

// Equivalent to:
function PersonFunction(name) {
  this.name = name;
}
PersonFunction.prototype.greet = function() { return `Hello, ${this.name}`; };
```

### **Objects in JavaScript**
- Everything is an object (almost) - functions, arrays, dates, etc.
- Can be created via:
  - Object literal: `{ prop: value }`
  - Constructor: `new Object()`, `new Date()`, etc.
  - `Object.create()`
  - Class instantiation: `new MyClass()`

## Practical Examples

### **Multiple Objects from One Class**
```javascript
class Employee {
  constructor(name, salary) {
    this.name = name;
    this.salary = salary;
  }

  getDetails() {
    return `${this.name} earns $${this.salary}`;
  }
}

// Creating multiple objects
const emp1 = new Employee('Alice', 50000);
const emp2 = new Employee('Bob', 60000);
const emp3 = new Employee('Charlie', 55000);

console.log(emp1.getDetails()); // 'Alice earns $50000'
console.log(emp2.getDetails()); // 'Bob earns $60000'
```

### **Object without Class**
```javascript
const employee1 = {
  name: 'Alice',
  salary: 50000,
  getDetails: function() {
    return `${this.name} earns $${this.salary}`;
  }
};

employee1.getDetails(); // 'Alice earns $50000'
```

## Object as a Supersets

All classes eventually create objects, but not all objects are necessarily created from classes. In JavaScript:

- **All user-defined classes are functions**, and function objects have the `Function` prototype
- **All objects (except primitives) inherit from `Object.prototype`**
- **You can create objects without any class involvement**

## When to Use Classes vs Objects

### **Use Classes When:**
- You need multiple similar objects
- You want to define reusable blueprints
- You need inheritance hierarchies
- You want to organize code into logical categories

### **Use Objects When:**
- You need a single, unique entity
- You're working with configuration objects
- You want flexibility in properties and methods
- You're dealing with data that doesn't fit class hierarchies

## Common Misconceptions

### **Everything is an Object**
While "everything is an object" is often repeated, there are **7 primitive types** in JavaScript:
```javascript
// Primitives (not objects):
const string = 'hello';      // string
const number = 42;           // number
const boolean = true;        // boolean
const undefined = undefined; // undefined
const null = null;           // null
const symbol = Symbol();     // symbol (ES6)
const bigInt = 123n;         // bigint (ES2020)

// Objects:
const object = {};           // object
const array = [];            // object
const date = new Date();     // object
```

### **Classes Create Objects**
Classes don't create anything - you need to call them with `new`:
```javascript
class MyClass {}       // Defines class (no object created)
const instance = new MyClass();  // Creates object
```

### **Objects Can Have Methods**
Objects can have methods, but they don't share them efficiently across instances:
```javascript
const obj1 = {
  name: 'Alice',
  greet: function() { return `Hi ${this.name}`; }
};

const obj2 = {
  name: 'Bob',
  greet: function() { return `Hi ${this.name}`; } // Duplicated method!
};

obj1.greet === obj2.greet; // false (different function instances)
```

Understanding this distinction helps you write more appropriate and efficient JavaScript code, whether you're using prototype-based or class-based patterns.
