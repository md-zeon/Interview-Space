# Why would you use default parameters?

Default parameters in JavaScript (introduced in ES6) allow you to specify default values for function parameters if no value or `undefined` is passed when the function is called. This helps make functions more flexible and robust by handling cases where arguments might be omitted.

## How They Work
- If no argument is provided for a parameter with a default value, the default is used
- If `undefined` is explicitly passed, the default is also used
- If any other value (including falsy values like `0`, `false`, `""`) is passed, the passed value is used
- Default parameters can reference other parameters that come before them in the function definition

## Benefits
1. **Eliminates Manual Checks**: No need to manually check `if (param === undefined)` inside the function body
2. **Cleaner Code**: Parameters with defaults don't need conditional logic for missing values
3. **Better API Design**: Makes it clear which parameters are optional when reading the function signature
4. **Reusability**: Functions can work with fewer arguments, making them more flexible
5. **Improved Readability**: Function signatures are more self-documenting

## Before ES6 (Workaround)
```javascript
function greet(name, message) {
    name = name || 'Anonymous'; // Logical OR fallback
    message = message !== undefined ? message : 'Hello!'; // Explicit undefined check
    return `${message} ${name}`;
}
```

## With ES6 Default Parameters
```javascript
function greet(name = 'Anonymous', message = 'Hello!') {
    return `${message} ${name}`;
}

// Usage examples:
greet(); // "Hello! Anonymous"
greet('John'); // "Hello! John"
greet('John', 'Hi there'); // "Hi there John"
greet(undefined, 'Yo'); // "Yo Anonymous" (undefined triggers default)
greet('Jane', undefined); // "Hello! Jane"
```

## Advanced Usage
```javascript
// Using functions as defaults
function getDefaultMessage() {
    return 'Default message';
}
function log(level, message = getDefaultMessage()) {
    console.log(`[${level}] ${message}`);
}

// Referencing other parameters
function createUser(name, email, displayName = name) {
    return { name, email, displayName };
}

createUser('John', 'john@example.com'); // { name: 'John', email: 'john@example.com', displayName: 'John' }

// Default parameters with destructuring
function setup(options = {}) {
    const { width = 800, height = 600, theme = 'light' } = options;
    return { width, height, theme };
}

setup(); // { width: 800, height: 600, theme: 'light' }
setup({ width: 1024 }); // { width: 1024, height: 600, theme: 'light' }
```

## Use Cases
- **Configuration Objects**: API functions that accept config objects with optional properties
- **Form Handling**: Functions that process form data with optional fields
- **Library Functions**: Making utility functions more user-friendly
- **Recursive Functions**: Providing base cases for recursive algorithms
- **Event Handlers**: Handling events with optional data

## Important Notes
- Default parameters are the same as explicitly passing `undefined`
- Default parameter expressions are evaluated at call time, not definition time
- They work well with rest parameters (`...args`)
- Can be combined with destructuring assignment
