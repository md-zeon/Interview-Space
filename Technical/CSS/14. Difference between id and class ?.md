# Difference between id and class in CSS

## Overview
Both `id` and `class` are attributes used in HTML to assign styles via CSS or for identification purposes. However, they have key differences in usage, specificity, and behavior.

## Key Differences

### 1. **Uniqueness**
- **ID**: Must be unique within an HTML document. Each `id` value can only be used once per page.
- **Class**: Can be reused multiple times within the same document. Multiple elements can share the same class.

### 2. **Syntax in HTML**
- **ID**: `<div id="unique-header">...</div>`
- **Class**: `<div class="button primary">...</div>` (can have multiple classes separated by spaces)

### 3. **CSS Selector**
- **ID**: Uses `#` prefix (e.g., `#unique-header`)
- **Class**: Uses `.` prefix (e.g., `.button`)

### 4. **Specificity**
- **ID**: Higher specificity (100 points in CSS specificity)
- **Class**: Lower specificity (10 points in CSS specificity)

### 5. **JavaScript Usage**
- **ID**: Faster for single element selection (document.getElementById())
- **Class**: Used for selecting multiple elements (document.getElementsByClassName())

### 6. **Purpose**
- **ID**: For unique element identification, styling, or as anchors for navigation
- **Class**: For styling multiple elements consistently

## Examples

### ID Example
```html
<div id="main-content">
  <p>This is the main content.</p>
</div>
```

```css
#main-content {
  width: 600px;
  margin: 0 auto;
}
```

### Class Example
```html
<button class="btn btn-primary">Save</button>
<button class="btn btn-secondary">Cancel</button>
<input class="form-input" type="text">
```

```css
.btn {
  padding: 10px 15px;
  border: none;
  border-radius: 4px;
}

.btn-primary {
  background-color: blue;
  color: white;
}
```

## Best Practices
- Use IDs for unique elements that need specific styles or JavaScript targeting
- Use classes for reusable styles across multiple elements
- Avoid overusing IDs for styling; prefer classes for consistency
- Keep specificity balanced to avoid !important declarations

## Note
In modern web development with JavaScript frameworks, the line between IDs and classes can blur, but the core principles remain the same.
