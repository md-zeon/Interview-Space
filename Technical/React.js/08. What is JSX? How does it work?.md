# What is JSX? How does it work?

JSX (JavaScript XML) is a syntax extension for JavaScript that allows you to write HTML-like code within JavaScript files. It's primarily used in React applications to describe what the UI should look like.

## What is JSX?
- **Full Name**: JavaScript XML
- **Purpose**: Enables writing HTML-like structures in JavaScript code without segregation
- **Integration**: Allows mixing HTML with JavaScript seamlessly
- **Not Valid Code**: JSX is not valid JavaScript or HTML; it needs to be transformed

## How JSX Works
JSX gets transformed into plain JavaScript function calls before the browser can execute it. The transformation process involves:

### 1. Writing JSX Code
You write a mix of HTML-like syntax and JavaScript expressions:
```jsx
const element = (
  <div className="container">
    <h1>Hello, {name}!</h1>
    <p>Welcome to React</p>
  </div>
);
```

### 2. JSX Transformation (Babel Transpilation)
The build tool (usually Babel) converts JSX into React.createElement() calls:
```jsx
const element = React.createElement(
  'div',
  { className: 'container' },
  React.createElement('h1', null, 'Hello, ', name, '!'),
  React.createElement('p', null, 'Welcome to React')
);
```

### 3. React Element Creation
- `React.createElement()` returns a plain JavaScript object (React element)
- No real DOM elements are created yet
- These objects describe what should be rendered

### 4. ReactDOM Rendering
Finally, this element is rendered to the DOM:
```jsx
ReactDOM.render(element, document.getElementById('root'));
```

## Key JSX Features and Rules
1. **Class vs className**: Use `className` instead of `class` for CSS classes
2. **JavaScript Expressions**: Use `{}` to embed any valid JavaScript expression
3. **Single Root Element**: JSX expressions must return one root element
4. **Self-Closing Tags**: Use `/` for self-closing tags (e.g., `<input />`, `<br />`)
5. **Component Names**: Must start with capital letter for custom components

## Example with Components and Logic
```jsx
function Greeting({ name, isGuest }) {
  return (
    <div>
      <h1 className="greeting">
        {isGuest ? 'Welcome, Guest!' : `Hello, ${name}!`}
      </h1>
      {!isGuest && <p>Great to see you back!</p>}
    </div>
  );
}

const user = { name: 'John', isGuest: false };
const greetingElement = <Greeting {...user} />;
```

JSX makes React components more readable and easier to write, bridging the gap between logic (JavaScript) and presentation (HTML) in a single, cohesive syntax.
