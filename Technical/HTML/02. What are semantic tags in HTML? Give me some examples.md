# What are semantic tags in HTML? Give me some examples.

## What are Semantic Tags?

Semantic HTML tags are elements that clearly describe their meaning to both the browser and the developer. Unlike generic tags like `<div>`, semantic tags provide context about the content they contain, making the code more readable and accessible.

## Benefits of Semantic Tags

1. **Accessibility**: Screen readers and assistive technologies can better understand the page structure
2. **SEO**: Search engines can better index and understand content
3. **Maintainability**: Code is easier to read and maintain
4. **Standards**: Following web standards and best practices

## Common Semantic Tags

### `<header>`
The `<header>` element represents introductory content for its nearest ancestor sectioning content or sectioning root element. Typically contains headings, logos, navigation, search forms, etc.

```html
<header>
  <h1>My Website</h1>
  <nav>
    <ul>
      <li><a href="#home">Home</a></li>
      <li><a href="#about">About</a></li>
    </ul>
  </nav>
</header>
```

### `<nav>`
The `<nav>` element represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents.

```html
<nav>
  <ul>
    <li><a href="/">Home</a></li>
    <li><a href="/products">Products</a></li>
    <li><a href="/contact">Contact</a></li>
  </ul>
</nav>
```

### `<main>`
The `<main>` element represents the main content of the body of a document or application. The main content area consists of content that is directly related to or expands upon the central topic of a document.

```html
<main>
  <article>
    <h2>Article Title</h2>
    <p>Main content goes here...</p>
  </article>
</main>
```

### `<section>`
The `<section>` element represents a standalone section of a document, which doesn't have a more specific semantic element to represent it.

```html
<section>
  <h2>News Section</h2>
  <article>
    <h3>News Item 1</h3>
    <p>Content...</p>
  </article>
</section>
```

### `<article>`
The `<article>` element represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable.

```html
<article>
  <h2>Blog Post Title</h2>
  <p>By <span>Author Name</span> on <time>2023-01-01</time></p>
  <p>Article content...</p>
</article>
```

### `<aside>`
The `<aside>` element represents a portion of a document whose content is only indirectly related to the document's main content.

```html
<aside>
  <h3>Related Links</h3>
  <ul>
    <li><a href="#link1">Link 1</a></li>
    <li><a href="#link2">Link 2</a></li>
  </ul>
</aside>
```

### `<footer>`
The `<footer>` element represents footer content for its nearest ancestor sectioning content or sectioning root element.

```html
<footer>
  <p>&copy; 2023 My Website. All rights reserved.</p>
  <nav>
    <a href="/privacy">Privacy Policy</a>
    <a href="/terms">Terms of Service</a>
  </nav>
</footer>
```

### `<figure>` and `<figcaption>`
The `<figure>` element represents self-contained content, potentially with an optional caption, which is specified using the `<figcaption>` element.

```html
<figure>
  <img src="image.jpg" alt="Description">
  <figcaption>Caption describing the image</figcaption>
</figure>
```

### `<time>`
The `<time>` element represents a specific period in time.

```html
<p>The event starts at <time datetime="2023-12-31T23:59">11:59 PM</time> on New Year's Eve.</p>
```

### `<mark>`
The `<mark>` element represents text which is marked or highlighted for reference or notation purposes.

```html
<p>This is <mark>important</mark> information that you should note.</p>
```

### `<summary>` and `<details>`
The `<details>` element creates a disclosure widget in which information is visible only when the widget is toggled into an "open" state. The `<summary>` element provides a summary, caption, or legend for the disclosure box.

```html
<details>
  <summary>Click to expand</summary>
  <p>Additional details that are hidden by default.</p>
</details>
```

### `<address>`
The `<address>` element indicates that the enclosed HTML provides contact information for a person or people, or for an organization.

```html
<address>
  <p>Written by <a href="mailto:webmaster@example.com">Jon Doe</a>.<br>
  Visit us at:<br>
  Example.com<br>
  Box 564, Disneyland<br>
  USA</p>
</address>
```

## Best Practices

1. Use semantic tags appropriately - don't force semantics where they don't belong
2. Combine semantic tags with proper headings hierarchy (h1-h6)
3. Avoid using semantic tags just for styling purposes
4. Test with screen readers to ensure accessibility
5. Maintain logical document outline

## Non-Semantic vs Semantic Example

### Non-Semantic (HTML4 style)
```html
<div id="header">
  <div class="nav">
    <!-- navigation -->
  </div>
</div>

<div class="main-content">
  <div class="article">
    <!-- article content -->
  </div>
</div>

<div id="footer">
  <!-- footer content -->
</div>
```

### Semantic (HTML5)
```html
<header>
  <nav>
    <!-- navigation -->
  </nav>
</header>

<main>
  <article>
    <!-- article content -->
  </article>
</main>

<footer>
  <!-- footer content -->
</footer>
```

Semantic tags make the HTML structure clearer and more meaningful, benefiting both developers and assistive technologies.
