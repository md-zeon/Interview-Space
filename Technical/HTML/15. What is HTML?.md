# What is HTML?

HTML (HyperText Markup Language) is the standard markup language used to create and structure web pages. It's the foundation of the World Wide Web and provides the basic building blocks for web content.

## What HTML Stands For

**H**yper**T**ext **M**arkup **L**anguage

- **HyperText**: Text that contains links to other texts
- **Markup**: Uses tags to annotate content
- **Language**: A system with its own syntax and rules

## Purpose of HTML

### 1. Structure Web Content

HTML provides the skeletal structure for web pages by defining elements like headings, paragraphs, lists, tables, and more.

### 2. Create Semantic Meaning

HTML elements give meaning to content - telling browsers and assistive technologies what different parts of a page represent.

### 3. Enable Interactivity

HTML works with CSS and JavaScript to create interactive, visually appealing websites.

### 4. Ensure Accessibility

Proper HTML structure makes websites accessible to people with disabilities and improves SEO.

### 5. Cross-Platform Compatibility

HTML works across different devices, browsers, and platforms.

## Basic HTML Structure

Every HTML document follows a basic structure:

```html
<!DOCTYPE html>
<html>
	<head>
		<title>My First Web Page</title>
	</head>
	<body>
		<h1>Hello World!</h1>
		<p>This is my first paragraph.</p>
	</body>
</html>
```

### Breaking Down the Structure

#### `<!DOCTYPE html>`

- Declares the document type and HTML version
- Must be the first line
- Tells browser to use standards mode

#### `<html>` (Root Element)

- Contains all other HTML elements
- Represents the root of the document

#### `<head>` (Document Head)

- Contains meta-information about the document
- Not displayed in the browser window
- Includes title, stylesheets, scripts, meta tags

#### `<title>`

- Sets the title shown in browser tab
- Important for SEO and bookmarks

#### `<body>` (Document Body)

- Contains visible page content
- Main content area where users interact

## HTML Elements and Tags

### What is an Element?

An HTML element is a complete structure that includes:

- Opening tag
- Content (optional)
- Closing tag (if not self-closing)

### What is a Tag?

Tags are the markup symbols that define elements:

- `<tagname>` - Opening tag
- `</tagname>` - Closing tag
- `<tagname />` - Self-closing tag

### Common HTML Elements

#### Headings (h1-h6)

```html
<h1>Main heading (most important)</h1>
<h2>Section heading</h2>
<h3>Subsection heading</h3>
<h4>Sub-subsection heading</h4>
<h5>Minor heading</h5>
<h6>Least important heading</h6>
```

#### Paragraphs

```html
<p>This is a paragraph of text.</p>
<p>Another paragraph with <em>emphasis</em>.</p>
```

#### Links

```html
<a href="https://www.example.com">Visit Example</a>
<a href="#section1">Jump to section</a>
```

#### Images

```html
<img
	src="image.jpg"
	alt="Description of image"
	width="300"
	height="200" />
```

#### Lists

```html
<!-- Unordered list -->
<ul>
	<li>First item</li>
	<li>Second item</li>
	<li>Third item</li>
</ul>

<!-- Ordered list -->
<ol>
	<li>First step</li>
	<li>Second step</li>
	<li>Third step</li>
</ol>
```

#### Containers

```html
<div class="container">
	<p>Content inside a div</p>
</div>

<section>
	<h2>Section with content</h2>
	<p>This is a semantic section.</p>
</section>
```

#### Forms

```html
<form
	action="/submit"
	method="post">
	<label for="name">Name:</label>
	<input
		type="text"
		id="name"
		name="name" />
	<button type="submit">Submit</button>
</form>
```

## HTML Versions

### HTML 1.0 (1993)

- Basic elements for text formatting
- First standardized version

### HTML 2.0 (1995)

- Added forms and input elements
- W3C took over standardization

### HTML 3.2 (1997)

- Tables, applets, superscripts
- More presentational elements

### HTML 4.0 (1997) and 4.01 (1999)

- Stylesheets, scripting
- Focus on semantic structure
- Strict, Transitional, and Frameset variants

### XHTML (2000)

- XML-based syntax
- Stricter rules
- Better compatibility

### HTML5 (2014)

- Current standard
- Semantic elements (`<article>`, `<section>`, `<nav>`)
- Native audio/video support
- Canvas for graphics
- Form enhancements
- API support (Geolocation, Local Storage)

## HTML Attributes

Attributes provide additional information about elements:

```html
<img
	src="photo.jpg"
	alt="A beautiful sunset"
	width="600"
	height="400"
	class="responsive-image" />
```

### Common Attributes

- `class` - CSS styling classes
- `id` - Unique identifier
- `src` - Source URL (images, scripts)
- `href` - Link destination
- `alt` - Alternative text for images
- `title` - Tooltip text
- `style` - Inline CSS styling
- `data-*` - Custom data attributes

## How HTML Works with CSS and JavaScript

### The Three Pillars of Web Development

#### HTML - Structure

```html
<div class="header">
	<h1 id="main-title">Welcome</h1>
	<nav>
		<a href="#about">About</a>
	</nav>
</div>
```

#### CSS - Presentation

```css
.header {
	background-color: #333;
	color: white;
	padding: 20px;
}

#main-title {
	font-size: 2em;
}
```

#### JavaScript - Behavior

```javascript
// Get the main title element
const title = document.getElementById("main-title");

// Change the text when clicked
title.addEventListener("click", function () {
	this.textContent = "You clicked the title!";
});
```

### Separation of Concerns

- **HTML**: Content structure and semantics
- **CSS**: Visual presentation and layout
- **JavaScript**: Interactive behavior and dynamic content

## Browser Rendering Process

1. **Parse HTML**: Browser reads and parses HTML code
2. **Create DOM**: Builds Document Object Model tree
3. **Load Resources**: Downloads CSS, images, scripts
4. **Apply Styles**: CSS is applied to HTML elements
5. **Execute Scripts**: JavaScript runs and modifies the DOM
6. **Render Page**: Final page is displayed to the user

## HTML Best Practices

### 1. Use Semantic HTML

```html
<!-- Good: semantic meaning -->
<header>
	<nav>
		<ul>
			<li><a href="/">Home</a></li>
		</ul>
	</nav>
</header>

<main>
	<article>
		<h1>Article Title</h1>
		<p>Content...</p>
	</article>
</main>

<footer>
	<p>&copy; 2023</p>
</footer>
```

### 2. Structure Content Logically

```html
<!-- Good: proper heading hierarchy -->
<h1>Main Title</h1>

<h2>Section One</h2>
<h3>Subsection</h3>

<h2>Section Two</h2>
```

### 3. Ensure Accessibility

```html
<!-- Alt text for images -->
<img
	src="logo.png"
	alt="Company Logo" />

<!-- Labels for forms -->
<label for="email">Email Address:</label>
<input
	type="email"
	id="email"
	name="email" />
```

### 4. Validate Your Code

Use HTML validators to check for errors and ensure standards compliance.

## HTML vs XHTML

### HTML (Forgiving)

```html
<p>Hello <em>world</p>
</em>
<img src="test.jpg">
<input type="text">
```

### XHTML (Strict)

```html
<p>Hello <em>world</em></p>
<img
	src="test.jpg"
	alt="Test" />
<input
	type="text"
	name="input" />
```

**XHTML Differences:**

- Must be well-formed XML
- All elements must be closed
- Self-closing tags end with `/>`
- Attribute values must be quoted
- Case-sensitive (lowercase required)

## Common HTML Mistakes

### 1. Missing DOCTYPE

```html
<!-- Wrong: no DOCTYPE -->
<html>
	<head>
		<title>Page</title>
	</head>

	<!-- Right: include DOCTYPE -->
	<!DOCTYPE html>
	<html>
		<head>
			<title>Page</title>
		</head>
	</html>
</html>
```

### 2. Unclosed Tags

```html
<!-- Wrong: missing closing tag -->
<p>
	Hello world!

	<!-- Right: properly closed -->
</p>

<p>Hello world!</p>
```

### 3. Invalid Nesting

```html
<!-- Wrong: invalid nesting -->
<p>This is <em>invalid</p> text</em>

<!-- Right: proper nesting -->
<p>This is <em>valid</em> text</p>
```

### 4. Missing Alt Text

```html
<!-- Wrong: no alt attribute -->
<img src="photo.jpg" />

<!-- Right: include alt text -->
<img
	src="photo.jpg"
	alt="A beautiful landscape" />
```
